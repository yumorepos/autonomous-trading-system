## Architecture Evolution

### Before (Level 2.0 — Reactive):
```
Scanner (cron, every 4h) → Position opens
   ↓
Watchdog (30-sec poll) → "Is guardian stale?"
   ↓
Guardian runs ONCE → checks SL → exits
   ↓
Guardian exits (not running anymore)
```
**Problems:** 30-sec gaps, dual authority, race conditions

### After (Level 2.95 — Production-safe):
```
Trading Engine (always-on, 1-sec loop):
   ├─ protect_capital() [force-mode SL, unblockable]
   ├─ periodic_reconciliation() [every 60 sec]
   ├─ scan_opportunities() [every 5 min, only if healthy]
   └─ update_heartbeat() [every cycle]
```
**Guarantees:** No 30-sec gaps, single authority, no races

## Bugs Found & Fixed

### Bug 1: Circuit Breaker Used Perps-Only Value
**File:** `scripts/risk-guardian.py` (now deprecated)
**Issue:** Used `account_value` (perps) instead of `account_value + spot_usd` (total)
**Result:** False 90% drawdown alerts blocked legitimate exits
**Fix:** Changed to total wallet value
**Status:** Fixed, then entire guardian deprecated for trading_engine.py

### Bug 2: Slippage Check Blocked SL Exits
**File:** `scripts/risk-guardian.py` (deprecated)
**Issue:** Compared current price to entry price, blocked SUPER exit at -23% because "slippage" was 7.6%
**Fix:** Skip slippage check for STOP_LOSS triggers (must exit at any price)
**Status:** Fixed in deprecated code, new engine has force-mode which bypasses all checks

### Bug 3: No Runtime Reconciliation
**File:** `scripts/trading_engine.py`
**Issue:** Engine only reconciled positions on startup, not during runtime. Manual closes went undetected for hours.
**Impact:** Stale state persisted, positions could be orphaned
**Fix:** Added `periodic_reconciliation()` (runs every 60 sec)
**Validation:** Tested with manual open/close, detected within 60 sec
**Status:** FIXED + TESTED

### Bug 4: Ledger Integration Error
**File:** `scripts/trading_engine.py`
**Issue:** `log_exit()` called with `pnl_usd` param, but TradeLogger doesn't accept it
**Fix:** Removed param (TradeLogger calculates PnL internally)
**Status:** FIXED

### Bug 5: Missing Price Fallback
**File:** `scripts/trading_engine.py`
**Issue:** If `get_mid()` returns 0.0 (price unavailable), exit logged with $0 price
**Fix:** Use entry_price as fallback if mid==0.0
**Status:** FIXED
